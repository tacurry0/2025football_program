# Jリーグ自動更新システム (GAS版) セットアップガイド

作成した `code.gs` は、Jリーグ公式サイトを定期的に解析し、あなたのアプリが必要とする最新の順位表や試合結果をJSON形式で配信する「ミニサーバー（Web API）」として機能します。

## 1. Google Apps Script の準備

1.  [Google Apps Script](https://script.google.com/home) にアクセスし、「新しいプロジェクト」を作成します。
2.  エディタに表示されている `function myFunction() { ... }` をすべて消去します。
3.  [code.gs](file:///c:/Users/takahashi/Downloads/trapp/gas/code.gs) の内容をすべてコピーして、エディタに貼り付けます。
4.  プロジェクトを保存します（例: `JLeagueDataAPI`）。

## 2. Webアプリとしてデプロイ（公開）

APIとして外部（アプリ）からアクセスできるように設定します。

1.  右上の「デプロイ」ボタンから **「新しいデプロイ」** を選択します。
2.  「種類を選択」の歯車アイコンから **「ウェブアプリ」** を選びます。
3.  設定を以下のようにします：
    *   **説明**: `Match Day App API`
    *   **次のユーザーとして実行**: `自分`
    *   **アクセスできるユーザー**: `全員`（※ログイン不要でアプリから読み込むため）
4.  「デプロイ」ボタンを押し、表示された **「ウェブアプリのURL」** をコピーしてメモしておきます。

## 3. 動作確認

ブラウザの新しいタブで、コピーしたURLの末尾にパラメータを付けてアクセスしてみてください。

*   **順位表の確認**: `[あなたのURL]?type=standings&league=j2`
*   **試合結果の確認**: `[あなたのURL]?type=results&league=j2`

成功すると、最新のデータが画面に表示されます。

## 4. アプリ（script.js）への統合（今後）

現在は `schedule.js` の固定データを使用していますが、今後このAPIのURLを `script.js` に設定することで、**「アプリを開くたびに最新スコアが反映される」** ようになります。

> [!IMPORTANT]
> このスクリプトはJリーグ公式サイトのHTML構造を解析しています。公式サイトのデザインが大幅にリニューアルされた場合、抽出ロジックの修正が必要になることがあります。

> [!TIP]
> 1時間ごとのキャッシュ機能を搭載しているため、頻繁にアクセスしても公式サイトに過度な負荷をかける心配はありません。
